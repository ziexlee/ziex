zieX는 단순한 채팅봇이 아닙니다. 코드 저장소를 읽고, Excel 데이터를 분석하며, 전문적인 문서와 차트를 생성하고, 명령줄을 실행합니다 — **모든 작업이 당신의 컴퓨터에서 실제로 수행됩니다**, 단지 "그런 것처럼 보이는" 것이 아닙니다.

이것은 스레드형 대화, 모델 설정, 스트리밍 출력, 워크스페이스 컨텍스트, 실제 도구 실행을 하나의 창에 통합한 Codex 스타일의 로컬 데스크톱 에이전트(macOS 네이티브 창 + 내장 로컬 서비스)이며, 내장된 Feishu(린크) 봇으로 언제 어디서나 한마디로 작업을 지시할 수 있습니다.

## 1. 핵심 기능

일상적인 업무부터 전문 개발까지, zieX는 하나의 에이전트 체계로 모든 생산성 시나리오를 다룹니다:

| 기능 | 설명 |
| --- | --- |
| **실제 코드 실행** | 단순한 채팅이 아닙니다. 워크스페이스에서 코드 파일을 읽고, 쓰고, 수정하며, 빌드와 테스트를 실행하는 완전한 개발 루프를 수행합니다. |
| **로컬 프라이버시 우선** | 에이전트는 당신의 컴퓨터에서 실행되며, 파일 조작과 명령 실행이 모두 로컬에서 이루어집니다. 코드와 데이터는 기기를 떠나지 않습니다. |
| **스마트 의도 라우팅** | 내장된 Planner, Review, Artifact 등 전문 에이전트가 의도를 자동으로 판단하고 작업을 분해하며, 최적의 도구 체인을 선택합니다. |
| **심층 파일 이해** | PDF, Word, Excel, PPT, CSV, 이미지, 코드 저장소 등 다양한 형식 분석을 지원합니다. Excel 자동 통계, 이미지 시각 인식, 코드 호출 체인 추적. |
| **멀티모달 창작** | 이미지, SVG 차트, 포스터, 문서 보고서, 교육 문제와 삽화를 생성합니다. 요구사항부터 완성품까지 한마디로. |
| **웹 리서치 기능** | 웹 검색과 URL 가져오기를 통합해 최신 정보, 시장 데이터, 기술 문서를 실시간으로 확보하고, 교차 검증으로 신뢰할 수 있는 분석을 제공합니다. |
| **Feishu 봇** | Feishu 대화에 연결해 봇을 @멘션하면 즉시 실행되고, 결과와 파일이 Feishu로 바로 전송됩니다. 모바일 업무에서도 로컬 에이전트를 구동할 수 있습니다. |

## 2. 워크플로우

zieX의 모든 작업은 "요구사항부터 완성품까지"의 폐쇄 루프를 따릅니다:

1. **요구사항 설명** — 자연어로 zieX에게 무엇을 하고 싶은지 알려줍니다: 코드 작성, 데이터 분석, 보고서 생성, 자료 검색 등 어떤 작업이든.
2. **에이전트 자동 계획** — Planner 에이전트가 의도를 분석하고, 단계를 분해하며, 가장 적합한 도구와 에이전트 분기를 선택합니다.
3. **실제 실행 및 검증** — 파일 읽기, 명령 실행, 웹 검색, 산출물 생성… 각 단계가 로컬에서 실제로 실행되고 자동 검증됩니다.
4. **완전한 결과 제공** — 코드 빌드 통과, 파일이 지정된 경로에 생성, 데이터 출처가 포함된 분석 — 초안이 아닌 완성품을 받습니다.

## 3. 에이전트 도구 모음

zieX의 "실행력"은 로컬에서 실제로 실행되는 도구 모음에서 나옵니다. 에이전트는 제한된 루프 안에서 결정 ↔ 관찰을 반복하며 하나씩 호출합니다:

- **파일**: `read_file` 읽기 · `write_file` 쓰기 · `inspect_file` 분석 · `analyze_spreadsheet` Excel 통계
- **명령**: `shell` 명령줄 실행 · `git_status` Git 상태 읽기
- **검색**: `search` 로컬 검색 · `web_search` 웹 검색 · `fetch_url` 페이지 가져오기
- **멀티모달**: `screenshot_page` 페이지 스크린샷 · `crop_image` 이미지 자르기 · `create_image` 이미지 생성
- **창작**: `create_document` 문서/보고서/포스터 · `create_image` 차트와 삽화
- **오케스트레이션 및 보안**: `parallel` 병렬 실행 · `permission` 권한 확인 · `snapshot` 쓰기 전 스냅샷 · `context` 컨텍스트 로드

> 모든 동작은 UI에서 실시간 "도구 카드"로 표시되어, 무엇을 하고 있는지 볼 수 있습니다.

## 4. 보안 및 방어

파일을 쓰고 명령을 실행할 수 있는 에이전트는 통제 가능해야 합니다. zieX에는 심층 방어가 내장되어 있습니다:

- **권한 등급 게이트**: `read / write / dangerousCommand / externalSend`별로 허용 또는 차단하며, 위험 명령은 기본적으로 거부합니다.
- **쓰기 전 스냅샷**: 매 쓰기/수정 전 자동 백업하며, 원클릭 롤백이 가능합니다.
- **워크스페이스 샌드박스**: 모든 경로 조작이 작업 디렉터리 내에 고정되며, 경계 침범이나 `..` 횡단을 금지합니다.
- **실행 후 자체 검증 + 제한된 복구**: 모델이 "완료"라고 한 후 산출물이 실제로 저장되었는지, 코드가 실제로 실행되는지 검증합니다. 불합격 시 제한된 횟수로 재시도하고 여전히 실패하면 정직하게 보고 — 결코 "완료된 척"하지 않습니다.
- **프라이버시 보호**: 출력은 키/민감 정보를 스캔하며, 설정과 키는 로컬에만 저장되고 zieX나 플랫폼 서버에 업로드되지 않습니다.

## 6. 사용법

### 1. 시작

```bash
cd /Users/zieli/Desktop/ziex
npm start          # Ziex.app 빌드 후 열기
```

`npm start`는 clang으로 네이티브 셸 `Ziex.app`을 컴파일하고 실행합니다. 앱은 로컬 서비스를 내장하며 주소는 `127.0.0.1:18765`입니다.

브라우저 모드(대체):

```bash
npm run server     # 로컬 서비스만 시작, 브라우저로 접속
```

### 2. 모델 설정

**설정 → 대형 모델 설정**을 열고, "추론 / 시각 이해 / 이미지 생성" 세 탭에 각각 입력:
- Full API URL(전체 엔드포인트)
- Model(모델명)
- API Key

Feishu 봇을 연결하려면 **Feishu 봇** 탭으로 전환해 App ID와 App Secret을 입력하세요.

### 3. 작업 시작

- 왼쪽 상단 **New chat**으로 스레드를 시작합니다;
- 하단 입력창에 자연어로 요구사항을 설명합니다, 예: "이 매출 Excel을 분석하고 차트가 포함된 주간 보고서 생성";
- `/`를 입력해 로컬 파일을 참조하고, `@`를 입력해 저장된 **스킬(Skills)**을 호출합니다;
- 전송 후 에이전트가 자동으로 계획, 실행, 검증, 제공합니다. 실행 중 **Stop**으로 중단할 수 있습니다.

### 4. 스킬(Skills)

스킬은 재사용 가능한 작업 템플릿입니다(예: 고정 형식의 주간 보고서, 고정된 코드 검토 흐름). 왼쪽 **skills** 메뉴에서 보기, 생성, 재사용이 가능해 반복 작업을 한마디로 해결합니다.

### 5. Feishu 봇

Feishu 앱에서 App ID / App Secret을 설정하면 대화 목록에 Feishu 봇 대화가 고정으로 나타납니다. Feishu에서 `@zieX`로 작업을 보내면 로컬에서 실제로 실행되고, 실행 결과와 생성된 파일이 Feishu로 바로 전송됩니다 — 출장, 출퇴근, 회의 중에도 에이전트를 구동하세요.
