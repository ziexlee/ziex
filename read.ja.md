zieXは単なるチャットボットではありません。コードリポジトリを読み、Excelデータを分析し、専門的な文書やグラフを生成し、コマンドラインを実行します — **すべての操作があなたのマシン上で実際に行われます**、単に「そう見える」だけではありません。

これは、スレッド型会話、モデル設定、ストリーミング出力、ワークスペースコンテキスト、実際のツール実行を一つのウィンドウに統合した Codex スタイルのローカルデスクトップエージェント(macOS ネイティブウィンドウ + 内蔵ローカルサービス)で、内蔵の Feishu(フィーシュー)ボットにより、いつでもどこでも一言で作業を指示できます。

## 1. コア機能

日常的なオフィス作業から専門的な開発まで、zieX は一つのエージェント体系であなたのあらゆる生産性シナリオをカバーします:

| 機能 | 説明 |
| --- | --- |
| **実コード実行** | 単なるチャットではありません。ワークスペース内でコードファイルの読み取り、書き込み、変更を行い、ビルドとテストを実行する完全な開発ループを実現します。 |
| **ローカル プライバシーファースト** | エージェントはあなたのマシン上で実行され、ファイル操作やコマンド実行はすべてローカルで完結します。コードやデータがデバイスから外に出ることはありません。 |
| **スマート意図ルーティング** | 内蔵の Planner、Review、Artifact などの専門エージェントが意図を自動判定し、タスクを分解して最適なツールチェーンを選択します。 |
| **深いファイル理解** | PDF、Word、Excel、PPT、CSV、画像、コードリポジトリなど多様な形式の分析をサポート。Excel 自動統計、画像の視覚認識、コード呼び出し連鎖の追跡。 |
| **マルチモーダル創作** | 画像、SVG グラフ、ポスター、文書レポート、教育問題と図を生成します。要件から完成品まで一言で。 |
| **ウェブリサーチ機能** | ウェブ検索と URL 取得を統合し、最新情報、市場データ、技術文書をリアルタイムで取得、クロス検証で信頼できる分析を提供します。 |
| **Feishu ボット** | Feishu の会話に接続し、ボットを @メンションすると即実行、結果とファイルが Feishu に直接送信されます。モバイルワークでもローカルエージェントを駆動できます。 |

## 2. ワークフロー

zieX のすべてのタスクは「要件から完成品まで」のクローズドループをたどります:

1. **要件を伝える** — 自然言語で zieX にやりたいことを伝えます: コード作成、データ分析、レポート生成、資料検索など、どんなタスクでも。
2. **エージェントが自動計画** — Planner エージェントが意図を分析し、ステップを分解し、最適なツールとエージェント分岐を選択します。
3. **実際の実行と検証** — ファイル読み取り、コマンド実行、ウェブ検索、成果物生成…各ステップがローカルで実際に実行され、自動検証されます。
4. **完全な成果物を提供** — コードのビルド通過、ファイルが指定パスに生成、データソース付きの分析 — 下書きではなく完成品を受け取れます。

## 3. エージェント ツールセット

zieX の「行動力」はローカルで実際に実行される一連のツールから来ます。エージェントは境界付きループの中で 決定 ↔ 観察 を繰り返し、一つずつ呼び出します:

- **ファイル系**: `read_file` 読み取り · `write_file` 書き込み · `inspect_file` 分析 · `analyze_spreadsheet` Excel 統計
- **コマンド系**: `shell` コマンドライン実行 · `git_status` Git 状態読み取り
- **検索系**: `search` ローカル検索 · `web_search` ウェブ検索 · `fetch_url` ページ取得
- **マルチモーダル系**: `screenshot_page` ページスクリーンショット · `crop_image` 画像切り抜き · `create_image` 画像生成
- **創作系**: `create_document` 文書/レポート/ポスター · `create_image` グラフと図
- **オーケストレーション・セキュリティ系**: `parallel` 並列実行 · `permission` 権限チェック · `snapshot` 書き込み前スナップショット · `context` コンテキスト読み込み

> すべてのアクションは UI 上でリアルタイムの「ツールカード」として表示され、何をしているかが見えます。

## 4. セキュリティと防護

ファイルを書き、コマンドを実行できるエージェントは制御可能でなければなりません。zieX には多重防御が内蔵されています:

- **権限レベルゲート**: `read / write / dangerousCommand / externalSend` ごとに許可またはブロックし、危険なコマンドはデフォルトで拒否します。
- **書き込み前スナップショット**: 毎回の書き込み/変更前に自動バックアップし、ワンクリックでロールバックできます。
- **ワークスペースサンドボックス**: すべてのパス操作を作業ディレクトリ内に固定し、境界越えや `..` トラバーサルを禁止します。
- **実行後自己検証 + 境界付きリカバリ**: モデルが「完了」と言った後、成果物が実際に保存されたか、コードが実際に動くかを検証します。不合格なら有限回リトライし、それでも失敗すれば正直に報告 — 決して「完了したふり」はしません。
- **プライバシー保護**: 出力はキー/機密情報をスキャンし、設定とキーはローカルにのみ保存、zieX やプラットフォームサーバーにアップロードされません。

## 5. 対応モデル

zieX は OpenAI 互換の `/chat/completions` エンドポイントに接続し、**推論 / 視覚理解 / 画像生成** の 3 つのモデルを個別に設定し、タスクタイプ別に自動ルーティングします。国産モデルプリセット内蔵:

- **DeepSeek**
- **通義千問 Qwen / DashScope 互換モード**
- **智譜 GLM**(GLM-4V 視覚、CogView テキスト→画像 を含む)
- **カスタム** — 任意の OpenAI 互換エンドポイント(セルフホスト、プライベートデプロイ対応)

## 6. 使い方

### 1. 起動

```bash
cd /Users/zieli/Desktop/ziex
npm start          # Ziex.app をビルドして開く
```

`npm start` は clang でネイティブシェル `Ziex.app` をコンパイルして起動します。アプリはローカルサービスを内蔵し、アドレスは `127.0.0.1:18765` です。

ブラウザモード(フォールバック):

```bash
npm run server     # ローカルサービスのみ起動、ブラウザでアクセス
```

### 2. モデル設定

**設定 → 大規模モデル設定** を開き、「推論 / 視覚理解 / 画像生成」の 3 つのタブにそれぞれ入力:
- Full API URL(完全なエンドポイント)
- Model(モデル名)
- API Key

Feishu ボットを接続するには **Feishu ボット** タブに切り替えて、App ID と App Secret を入力してください。

### 3. タスク開始

- 左上の **New chat** でスレッドを開始します;
- 下部の入力ボックスに自然言語で要件を記述します、例: 「この売上 Excel を分析してグラフ付きの週次レポートを生成」;
- `/` を入力してローカルファイルを参照し、`@` を入力して保存済みの **スキル(Skills)** を呼び出します;
- 送信後、エージェントが自動的に計画、実行、検証、提供します。実行中は **Stop** で中断できます。

### 4. スキル(Skills)

スキルは再利用可能なタスクテンプレートです(例: 固定フォーマットの週次レポート、固定のコードレビューフロー)。左側の **skills** メニューで確認、作成、再利用ができ、繰り返しの作業を一言で済ませます。

### 5. Feishu ボット

Feishu アプリで App ID / App Secret を設定すると、会話リストに Feishu ボットの会話が固定で表示されます。Feishu で `@zieX` してタスクを送るとローカルで実際に実行され、実行結果と生成ファイルが Feishu に直接送信されます — 出張、通勤、会議中でもエージェントを駆動できます。
